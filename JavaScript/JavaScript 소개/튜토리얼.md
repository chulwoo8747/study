## 코드의 실행

---

 기본적으로 JavaScript는 세미콜론(**;**)으로 구분된 구문 단위로 위에서부터 차례대로 실행된다. 그러나 제어 흐름, 예외 처리, 함수 호출 등을 만나면 코드의 실행 흐름이 이리저리 옮겨다니기도 한다.

JavaScript는 REPL이라는 툴을 이용해서 실행시킬 수 있다. JavaScript 예제 코드를 실행시키려면 repl.it을 이용할 수 있다.

## 기본 문법

---

### 대소문자의 구분

JavaScript는 모든 부분에서 대소문자 구분을 하기 때문에 주의해야 한다.

ex) `function`과 `Function`은 완전히 다른 의미를 가진다.

<br/>

### 세미콜론

JavaScript는 세미콜론(;)을 통해 각 구문을 구분한다.

```jsx
const a = 1;
const b = 2;
```

특별한 경우가 아니면, 세미콜론을 쓰고 나서는 줄바꿈을 해줘서 구문이 구분되어 보이게 하는것이 좋다.

<br/>

### 공백

```jsx
const a = 1;

const      x         =       1;

const
        x
            =
                1;
// 위 코드들은 똑같은 동작을 한다.
```

코드의 동작 여부와 별개로, 코드를 읽기 쉽도록 공백을 깔끔하게 유지해야 한다.

## 값(value)과 그 리터럴(literal)

---

프로그래밍은 ‘값’을 다루는 행위라 할 수 있다. JavaScript에도 여러 가지 값을 표현하는 문법이 있다. 이를 리터럴(literal)이라 한다.

```jsx
1 // 정수
2.5 // 실수
'hello world' // 작은 따옴표 문자열
"JavaScript" // 큰 따옴표 문자열
true // 참을 나타내는 진리값
false // 거짓을 나타내는 진리값
null // 존재하지 않음을 나타내는 값
undefined // 정의된 적 없음을 나타내는 값
```

<br/> 

## 값의 타입(type)

---

JavaScript에서 모든 값은 그 ‘타입을’ 가지고 있다. 쉽게 ‘값의 종류’라고 보면 된다. `typeof` 연산자는 피연산자의 타입을 가리키는 문자열을 반환한다.

```jsx
typeof 1 // number
typeof 2.5 // number
typeof 'hello world' // string
typeof true // boolean
typeof null // object
typeof 
```

## 표현식(expression)과 연산자(operator)

---

표현식이란 **값으로 변환될 수 있는 부분**을 가리킨다.

JavaScript에서의 연산자는 수학에서 보던 것과는 동작 방식이 달라서 입문자에게는 익숙하지 않은 연산자들도 있다.

```jsx
// 사칙 연산
1 + 3;
2 - 4;
3 * 5;
4 / 6;

// 연산자 우선순위 (operator precedence)
1 + 2 * 3;
(1 + 2) * 3;

// 논리 연산
true || false;
true && false;

// 진리값으로 취급하기
1 || 0;
1 && 0;
```

## 변수(variable)

---

값을 재사용하기 위해 값에 붙일 이름을 선언하고 그 이름에 값을 대입할 수 있다. 이 때 값의 이름을 변수(variable)라고 부른다. 

```jsx
//한 변수의 선언
let v;

// 값의 대입
v = 1;

// 변수의 선언과 값의 대입을 동시에
let x = 1;

// 두 변수의 선언과 대입
let y = 2, z = x + y;

//const 변수
const c = 1;
c = 2; // ERROR!

//선언한 변수를 사용하기
x;
typeof x;
x * z; 
```

## 함수 (function)

---

값 뿐만 아니라 **코드 뭉치(?)**에 이름을 붙여서 재사용을 할 수도 있는데, 이름 함수라고 한다. JavaScript에는 함수를 선언할 수 있는 여러 가지 방법이 있다.

```jsx
// `function` 키워드를 이용한 함수 선언
function add(x,y) {
    return x + y;
}

// arrow function
const multiply = (x,y) => x * y;

// 함수 호출하기
add(1,2); // 결과값은 3이다.
mulitply(3,4); // 결과값은 12이다.
```

## 객체

---

대부분의 프로그래밍 언어는 여러 개의 값을 한꺼번에 담아 통(container)처럼 사용할 수 있는 자료구조를 내장하고 있다.  JavaScript에는 **객체(object)**라는 자료구조가 있다.

객체에는 이름에 값이 연결되어 저장된다. 이를 객체의 **속성(property)**이라고 한다. 객체를 사용할 때는 속성 이름을 이용해서, 속성 값을 읽거나 쓸 수 있다.

```jsx
// 객체의 생성
const obj = {
    x: 0, // 객체의 속성. 속성 이름: x, 속성 값: 0
    y: 1; // 객체의 속성. 속성 이름: y, 속성 값: 1
}

// 객체의 속성에 접근하기
obj. x;
obj['y'];

// 객체의 속성 변경하기
obj.x += 1;
obj['y'] -=1;

// 객체의 속성 삭제하기
delate obj.x;
```

객체의 속성을 통해 사용하는 함수를 **메소드(method)**라고 부른다. 

```jsx
const obj = {
    x: 0,
    increaseX: function() {
        this.x = this.x + 1;
    }
};

obj.increaseX();
console.log(obj.x); // 1
```

## 배열

---

JavaScript에서의 배열은 배열에 담은 데이터는 객체의 경우와 달리 **요소(element)** 혹은 **항목(item)**이라고 부른다. 객체와 마찬가지로 배열 안에 여러 개의 값을 저장할 수 있지만, 배열 요소 간에는 **순서**가 존재하며, 이름 대신에 **인덱스(index)를** 이용해 값에 접근한다.

```jsx
// 배열의 생성
const arr = ['one', 'two', 'three'];

//인덱스를 사용해 배열의 요소(element)에 접근할 수 있다.
//배열 인덱스(index)는 0 부터 시작한다.
arr[0]; // === 'one'
arr[1]; // === 'two'

// 여러 타입의 값이 들어있는 배열
[1, 2, 3, 'a', 'b', {x: 0, y: 0, name: '원점'}];

// 배열에 요소 추가하기
arr.push('four);

// 배열의 요소 삭제하기
arr.splice(3,1); // 인덱스가 3인 요소부터 시작해서 하나를 삭제한다.
```
